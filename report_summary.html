[SentinelOne] Cloud-native architecture with no on-premises deployment option
SentinelOne is fundamentally a cloud SaaS platform. All endpoint telemetry and threat data is transmitted to SentinelOne's cloud-hosted Management Console. Organizations with strict data sovereignty requirements (e.g., air-gapped networks, classified environments) cannot deploy the full platform locally. The agent does operate offline with autonomous protection, but data flows to cloud when connectivity is restored.
[SentinelOne] Endpoint metadata and threat data used in ML model training
The privacy policy explicitly states that Threat Data (including malware, URLs, processes, techniques, metadata) is used in SentinelOne's proprietary machine learning engines. While limited personal information is stated as not being incorporated into the Solutions, the use of customer-derived threat data for ML training may conflict with strict data isolation requirements in regulated industries.
[Okta Workforce Identity] Cloud-only architecture with no full on-premises option
Okta is a cloud-native SaaS platform. While Okta Access Gateway provides hybrid AD bridge capabilities, the core IAM service runs exclusively in Okta's cloud. Organizations with strict data sovereignty requirements requiring fully on-premises IAM cannot use Okta directly.
[Microsoft Entra ID] Privileged Identity Management requires P2 licensing
PIM, Access Reviews, and Identity Protection are only available with Entra ID P2 licenses. Organizations on Free or P1 tiers lack just-in-time privileged access controls, creating standing privilege risk.
[Fortinet FortiGate] Historical critical CVEs in FortiOS SSL VPN and management plane
FortiOS has experienced critical remotely exploitable vulnerabilities (e.g., CVE-2024-21762 heap overflow in SSL VPN, CVE-2023-27997 IPSec VPN RCE, CVE-2022-42475 SSL VPN RCE). Organizations must maintain aggressive patching schedules and follow Fortinet PSIRT advisories. SSL VPN should be carefully hardened per documented best practices.
[GitHub Copilot Enterprise] Code snippets transmitted to cloud for inference
Copilot sends code context (surrounding lines, open files) to GitHub/Microsoft cloud for generating suggestions. While Enterprise plan promises no training on customer data, the code is still transmitted to and temporarily processed in Microsoft's cloud infrastructure. This creates data exposure risk for organizations handling highly sensitive or classified code.
[GitHub Copilot Enterprise] Limited prompt injection defense for suggestion engine
Copilot's suggestion engine lacks documented defenses against prompt injection via malicious code in workspace files or dependencies. An attacker who compromises a dependency or injects malicious comments in code could potentially influence Copilot's suggestions to include vulnerable patterns.
[Windsurf (Codeium)] Cascade agent executes commands without sandboxing
Windsurf's Cascade agent can execute terminal commands directly in the user's environment without sandboxing. A compromised dependency or malicious prompt injection could lead to arbitrary command execution on the developer's machine, potentially exfiltrating code or credentials.
[Windsurf (Codeium)] Significant prompt injection attack surface
Cascade reads entire codebase contexts including comments, dependencies, and configuration files. Malicious content in any of these could influence the agent's behavior. No documented defense system against prompt injection attacks.
