⚠️ 风险清单

HIGH Privileged Identity Management requires P2 licensing D2
PIM, Access Reviews, and Identity Protection are only available with Entra ID P2 licenses. Organizations on Free or P1 tiers lack just-in-time privileged access controls, creating standing privilege risk.
MEDIUM Conditional Access policy misconfiguration risk D7
Complex Conditional Access policies with multiple conditions and overlapping assignments can produce unintended access outcomes. While the What If tool helps, there is no automated policy conflict detection.
MEDIUM Third-party application consent exposure D6
The OAuth consent framework allows users to grant permissions to third-party applications. Without strict admin consent policies, users may authorize malicious or over-privileged applications accessing directory data.
MEDIUM Legacy authentication protocol risk D1
While Conditional Access can block legacy authentication, Exchange ActiveSync and other legacy protocols do not support MFA. Organizations that still require legacy protocols face a security gap.
LOW AI optimization agent preview status D5
The Conditional Access optimization agent is in preview, and its AI-driven policy recommendations may evolve. Organizations should review agent suggestions before adoption to ensure alignment with security policies.
LOW Cost escalation with premium features D8
Critical security features including PIM, Identity Protection, and advanced Conditional Access require P2 licensing. At enterprise scale, per-user P2 costs can become significant, potentially leading organizations to deprioritize security controls.

D7
Adopt Conditional Access templates for common security scenarios (require MFA for admins, block legacy auth, require compliant device). Use the What If tool to test all new policies before enabling them in report-only mode, then transition to enforce.
