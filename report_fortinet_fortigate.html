HIGH Historical critical CVEs in FortiOS SSL VPN and management plane D8
FortiOS has experienced critical remotely exploitable vulnerabilities (e.g., CVE-2024-21762 heap overflow in SSL VPN, CVE-2023-27997 IPSec VPN RCE, CVE-2022-42475 SSL VPN RCE). Organizations must maintain aggressive patching schedules and follow Fortinet PSIRT advisories. SSL VPN should be carefully hardened per documented best practices.
MEDIUM FortiGuard telemetry data transmission to Fortinet infrastructure D5
FortiGate communicates with FortiGuard Distribution Network for AV/IPS updates, web filtering lookups, and sandbox submissions. While air-gap mode is supported, default configurations transmit query data (including URLs and file hashes) to Fortinet servers. Organizations with strict data sovereignty requirements should evaluate FortiManager as a local FortiGuard server and minimize external queries.
MEDIUM CCPA-defined 'sales' of personal information by Fortinet D5
Fortinet's privacy policy acknowledges CCPA-defined 'sales' and 'sharing' of identifiers, internet activity, and commercial information to marketing and analytics providers. While this applies to Fortinet's corporate operations rather than FortiGate traffic data, organizations subject to strict privacy regulations should review Fortinet's data handling for FortiCloud and FortiGuard cloud services.
LOW Ecosystem lock-in to Fortinet Security Fabric D9
While Fortinet Security Fabric provides extensive integration across the Fortinet product portfolio, integration with third-party security tools is limited compared to more open platforms. Organizations with heterogeneous security environments should evaluate REST API and syslog integration capabilities for non-Fortinet tools.
LOW No FIDO2/WebAuthn support for admin console authentication D1
FortiGate supports MFA via FortiToken (TOTP, push notifications, hardware tokens) but does not natively support FIDO2/WebAuthn hardware security keys for direct admin console login. Organizations standardizing on YubiKey or similar FIDO2 tokens should use SAML SSO through an IdP that supports FIDO2 as a workaround.
